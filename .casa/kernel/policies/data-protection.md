# Data Protection Policy

## Required

- Classify personal, tenant, financial and credential data.
- Minimize collection and retention.
- Encrypt sensitive data in transit and at rest when supported by the stack.
- Restrict access with least privilege.
- Avoid logging sensitive data.
- Define deletion and export behavior for user data.

## Forbidden

- Storing secrets in source control.
- Using production personal data in local development without approval.
- Exposing tenant data across authorization boundaries.
