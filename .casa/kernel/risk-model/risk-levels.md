# Risk Levels

## Low Risk

Examples:
- documentation change
- isolated UI copy change
- non-critical visual adjustment

Approval:
- no special approval

## Medium Risk

Examples:
- new API endpoint
- database query change
- feature behavior change

Approval:
- reviewer recommended

## High Risk

Examples:
- authentication
- authorization
- secrets
- production infrastructure
- database migrations
- sensitive data

Approval:
- human approval required
- tests required
- rollback plan required
