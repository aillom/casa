# Legacy Standard

## Required

- Discover before changing.
- Preserve business behavior before refactoring.
- Add characterization or smoke tests where possible.
- Use seams, facades or anti-corruption layers for incremental change.
- Retire old code only after evidence and rollback planning.

## Avoid

- Rewrite-first modernization.
- Changing unclear code without dependency mapping.
- Removing old paths before traffic and behavior are verified.
