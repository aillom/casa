# Quality Gate: Security Check

## Purpose

Validate sensitive changes before completion.

## Applies When

- Auth, permissions, secrets, sensitive data, public APIs, logs, uploads or infrastructure change.

## Check

- Server-side authorization is explicit.
- External input is validated.
- Secrets and sensitive data are not logged.
- Errors do not expose internals.
- Required tests and approvals are present.
