# Quality Gate: API Contract Check

## Purpose

Validate public or shared API changes.

## Applies When

- Endpoints, DTOs, OpenAPI files or generated clients change.

## Check

- Route versioning is clear.
- Auth status is documented.
- Request and response DTOs are updated.
- Standard error model is preserved.
- List endpoints include pagination.
- Contract or integration tests cover the change.
