# Quality Gate: Legacy Safety Check

## Purpose

Validate modernization work before changing or retiring legacy behavior.

## Applies When

- Legacy, unclear or poorly tested modules change.
- A seam, facade, strangler path or retirement is introduced.

## Check

- Existing behavior is mapped.
- Characterization or smoke tests exist where practical.
- Candidate seams are documented.
- Rollback is possible for risky changes.
- Retirement only happens after evidence.
