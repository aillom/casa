# Repository Modules

## Core

- `.casa/kernel`: principles, standards, policies and risk model.
- `.casa/capabilities`: reusable skills, subagents and workflows.
- `.casa/mission-control`: templates for missions, evidence, handoffs, context capsules and risk gates.
- `.casa/context-capsules`: reusable scoped context packs for common mission types.
- `.casa/specs`: templates for feature, stack composition, API, security and brownfield specifications.
- `.casa/governance`: permissions, sensors, evals and audit guidance.
- `.casa/quality-gates`: versioned checks for architecture, security, API, UI and legacy safety.
- `.casa/registry`: indexes of skills, skill marketplace config, skill lockfile, agents, adapters, workflows, policies, stack packs and terminal recipes.
- `.casa/cockpit`: visual control-plane screen and information architecture docs.
- `.casa/context`: maps that help agents understand the repository without rediscovery.
- `.casa/modernization`: brownfield modernization playbooks.

## Generated and adapter outputs

- `.codex`: generated Codex skills.
- `.cursor`: generated Cursor rules.
- `.agents`: generated generic agent guide.
- `.casa/generated`: generated portable adapter packs and indexes.
- `.casa/runtime/missions`: optional mission records created during execution.
- `.casa/runtime/stack-plans`: optional stack plans generated by `casa compose` and `casa stack add`.
- `.casa/runtime/recipes`: optional recipe plans generated by `casa recipe`.
- `.casa/runtime/history`: local JSONL history for harness actions.

## Human-facing material

- `README.md`: public overview.
- `docs`: short introduction and mode documentation.
- `examples`: practical usage examples.
- `templates`: starter structures.
- `CASA_Agent_Native_Architecture_Manual.pdf`: long-form method manual.
