# Security Policy

C.A.S.A includes security policies, agent permissions and DevSecOps guidance.

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Report security concerns privately to the maintainers.

## Scope

Security reports may include:

- unsafe generated agent instructions
- insecure templates
- CI/CD security flaws
- dependency risks
- secrets handling issues
- infrastructure security issues
- authentication or authorization weaknesses in examples

## Security principles

C.A.S.A follows:

- least privilege
- secure defaults
- explicit authorization
- no secrets in Git
- defense in depth
- human approval for high-risk changes
- automated validation where possible
