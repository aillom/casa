# Contributing to C.A.S.A

Thanks for your interest in contributing to C.A.S.A.

C.A.S.A is an agent-native architecture method for building, governing and modernizing software systems.

## How to contribute

You can contribute by:

- improving documentation
- proposing new skills
- improving policies
- adding adapters for coding agents
- creating templates
- testing C.A.S.A in real projects
- reporting unclear parts of the method

## Contribution rules

Before opening a pull request:

1. Read `AGENTS.md`.
2. Check `casa.manifest.yaml`.
3. Follow the C.A.S.A structure.
4. Do not edit generated files directly.
5. Prefer small, focused pull requests.
6. Include rationale for architectural changes.
7. Update documentation when behavior or structure changes.

## Pull request quality bar

A good PR should explain:

- what changed
- why it changed
- what risks exist
- how it was tested
- which C.A.S.A area is affected

## Areas

- `kernel`: principles, standards, policies and risk model
- `context`: repo maps, architecture maps and legacy inventory
- `capabilities`: skills, subagents and workflows
- `protocols`: AGENTS.md, skills, MCP, A2A, OpenAPI and AsyncAPI
- `adapters`: tool-specific templates
- `modernization`: legacy modernization playbooks
- `governance`: permissions, sensors, evals and audit
