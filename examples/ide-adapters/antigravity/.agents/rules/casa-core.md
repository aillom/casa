# Rule: C.A.S.A Core

This project follows C.A.S.A - Context, Architecture, Stack & Automation.

## Always do

- Read `AGENTS.md` and `casa.manifest.yaml`.
- Treat `.casa` as the source of truth.
- Use specs before implementation.
- Follow policies and protected paths before sensitive changes.
- Do not edit generated adapter files directly.
- Run `npm run check` before completion when shell access is available.

## Why

C.A.S.A is the architecture layer for governed vibe coding: fast agent work with stable context, boundaries and validation.
