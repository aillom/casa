# Workflow: Implement Feature With C.A.S.A

## Goal

Use this as a workflow card for agents that support task workflows.

## Steps

1. Read `AGENTS.md`.
2. Read `casa.manifest.yaml`.
3. Read the feature spec.
4. Select the smallest useful capability set.
5. Identify risk level and protected paths.
6. Implement the smallest coherent change.
7. Update tests, docs, examples and contracts when needed.
8. Run `npm run check`.
9. Report evidence and remaining risks.
