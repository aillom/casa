# C.A.S.A Workspace Rule

Use this content in a workspace rule surface for IDE agents that support project-level instructions.

## Rule

This project follows C.A.S.A.

- Read `AGENTS.md` first.
- Read `casa.manifest.yaml` for modes, adapters, governance and protected paths.
- Use `.casa/specs` for requirements.
- Use `.casa/kernel/policies` for security and governance constraints.
- Do not edit generated adapter files directly.
- Run `npm run check` before completion when commands are available.
