# Devin Knowledge: C.A.S.A Brownfield Work

## When to use

Use before modifying legacy, unclear or poorly tested modules.

## Workflow

1. Run read-only discovery.
2. Map entry points and dependencies.
3. Identify behavior to preserve.
4. Add characterization or smoke tests where possible.
5. Create seams before refactoring.
6. Retire old code only after evidence and rollback planning.
