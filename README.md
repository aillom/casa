# C.A.S.A — Context, Architecture, Stack & Automation

C.A.S.A is an agent-native architecture layer for building, governing and modernizing software systems with AI coding agents.

It helps teams use Cursor, Codex, Claude, Devin, Antigravity and future agents through a protocol-first, adapter-driven architecture.

## Core idea

**Design for agents. Govern for humans. Evolve for legacy.**

## Why C.A.S.A?

AI coding agents are powerful, but without structure they create duplicated patterns, architecture drift, weak documentation, fragile automation and inconsistent software.

C.A.S.A turns vibe coding into governed software delivery.

## Pillars

- **Context**: rules, specs, policies, docs, repository maps and agent instructions.
- **Architecture**: boundaries, clean code, contracts, modular design and legacy modernization paths.
- **Stack**: recommended frontend, backend, database, DevOps, security and documentation choices.
- **Automation**: tests, CI/CD, OpenAPI, security checks, documentation, agent adapters and drift detection.

## Modes

- **Greenfield Mode**: start new projects with C.A.S.A from day one.
- **Brownfield Mode**: install C.A.S.A as an overlay on legacy systems without forcing immediate rewrites.

## Repository map

- `.casa/kernel` — source of truth for principles, standards, policies and risk model.
- `.casa/context` — repo maps, architecture maps, domain maps and legacy inventory.
- `.casa/capabilities` — skills, subagents, workflows and hooks.
- `.casa/protocols` — AGENTS.md, skills, MCP, A2A, OpenAPI and AsyncAPI alignment.
- `.casa/adapters` — templates for agent-specific integrations.
- `.casa/generated` — generated files for agents. Do not edit manually.
- `.casa/specs` — specification templates.
- `.casa/modernization` — legacy discovery, seams, strangler and retirement playbooks.
- `.casa/governance` — permissions, sensors, evals and audit model.
- `templates` — reusable project templates.
- `examples` — practical usage examples.
- `docs` — human-readable documentation.
